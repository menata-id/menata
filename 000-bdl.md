# Business Definition Language (BDL)

Version

0.1 Draft

Status

Draft

---

# Purpose

Business Definition Language (BDL) adalah bahasa deklaratif untuk mendeskripsikan pengetahuan bisnis secara terstruktur.

BDL memungkinkan aplikasi didefinisikan menggunakan metadata yang dapat dipahami oleh manusia, AI, maupun Runtime.

BDL tidak bergantung pada bahasa pemrograman, database, framework, maupun teknologi tertentu.

---

# Philosophy

Business berubah.

Teknologi berubah.

Framework berubah.

Bahasa pemrograman berubah.

Namun pengetahuan bisnis tetap memiliki makna.

BDL memisahkan pengetahuan bisnis dari implementasi teknis sehingga aplikasi dapat terus berkembang tanpa harus menulis ulang logika bisnis.

---

# Vision

Mendefinisikan aplikasi bisnis menggunakan bahasa yang berorientasi pada bisnis, bukan pada teknologi.

Business Analyst, Domain Expert, AI, dan Developer dapat bekerja menggunakan definisi yang sama.

---

# Principles

BDL dibangun berdasarkan prinsip-prinsip berikut.

- Business First
- Metadata First
- Knowledge Driven
- Declarative
- Human Readable
- Machine Readable
- AI Friendly
- Runtime Independent
- Storage Independent
- Renderer Independent
- Extensible
- Composable
- Backward Compatible

---

# Scope

BDL mendefinisikan struktur dan perilaku bisnis.

BDL tidak mendefinisikan implementasi teknis.

---

# Core Concepts

BDL dibangun dari sekumpulan Grammar.

Setiap Grammar memiliki satu tanggung jawab utama.

| Grammar | Menjawab Pertanyaan |
|----------|---------------------|
| Object | Apa konsep bisnisnya? |
| Field | Informasi apa yang diketahui tentang konsep tersebut? |
| Relationship | Bagaimana konsep-konsep tersebut saling berhubungan? |
| Action | Apa yang dapat dilakukan terhadap konsep tersebut? |
| Interaction | Bagaimana Action dijalankan? |
| Workflow | Bagaimana siklus hidup konsep tersebut berubah? |
| Rule | Aturan bisnis apa yang berlaku? |
| View | Bagaimana informasi disajikan? |
| Dashboard | Bagaimana kondisi bisnis dipantau? |
| Permission | Siapa yang boleh melakukan apa? |
| Automation | Apa yang dapat dijalankan secara otomatis? |
| Policy | Kebijakan apa yang membatasi perilaku sistem? |
| Recipe | Bagaimana sekumpulan Grammar dikemas menjadi solusi? |
| Workspace | Di mana solusi dijalankan? |
| Application | Bagaimana seluruh Grammar membentuk sebuah aplikasi? |

---

# Grammar Model

Setiap Grammar memiliki satu tanggung jawab.

Grammar tidak saling menggantikan.

Grammar saling melengkapi.

Hubungan antar Grammar.

```text
Object
    ├── Field
    ├── Relationship
    │
    ├── Action
    │      └── Interaction
    │
    ├── Workflow
    ├── Rule
    ├── View
    ├── Dashboard
    ├── Permission
    ├── Automation
    └── Policy

Recipe
    └── Menggabungkan Grammar menjadi solusi

Workspace
    └── Menjalankan Recipe

Application
    └── Menggabungkan seluruh Grammar menjadi sebuah aplikasi
```

# Layered Architecture

Business OS dipisahkan menjadi beberapa lapisan.

```text
Business Definition Language (BDL)
            │
            ▼
Runtime
            │
            ▼
Renderer
            │
            ▼
Deployment
```

Setiap lapisan mempunyai tanggung jawab yang berbeda.

---

# Separation of Concerns

BDL hanya mendeskripsikan pengetahuan bisnis.

Runtime menjalankan BDL.

Renderer menyajikan pengalaman pengguna.

Deployment mengelola lingkungan operasional.

AI membantu menghasilkan dan menyempurnakan BDL.

Setiap lapisan dapat berkembang secara independen.

---

# Normative Requirements

Seluruh Grammar BDL:

MUST mempunyai identitas.

MUST mempunyai tujuan yang jelas.

MUST menggunakan istilah bisnis.

MUST bebas dari implementasi teknis.

MUST dapat dipahami oleh pengguna bisnis.

MUST dapat diproses oleh Runtime.

SHOULD dapat digunakan kembali pada berbagai aplikasi.

SHOULD dapat dihasilkan maupun disempurnakan oleh AI.

---

# Non Goals

BDL tidak mendefinisikan:

- Bahasa pemrograman.
- Struktur database.
- ORM.
- Framework web.
- API.
- User Interface.
- HTML.
- CSS.
- JavaScript.
- Infrastruktur.
- Deployment.

Seluruh implementasi tersebut berada di luar ruang lingkup BDL.

---

# Example

Contoh sederhana.

```text
Object Customer

Field Customer Name

Field Email Address

Action Register Customer

Interaction Customer Registration

Workflow Customer Lifecycle

Rule Validate Email

View Customer List
```

Contoh tersebut hanya mendeskripsikan pengetahuan bisnis.

Cara aplikasi dijalankan ditentukan oleh Runtime.

---

# Compatibility

Grammar BDL harus tetap stabil meskipun Runtime, Renderer, AI, maupun teknologi implementasi berubah.

---

# Glossary

Grammar

Unit penyusun Business Definition Language.

Object

Representasi formal suatu konsep bisnis.

Metadata

Representasi deklaratif yang mendeskripsikan pengetahuan bisnis.

Runtime

Komponen yang menjalankan BDL.

Renderer

Komponen yang menyajikan BDL kepada pengguna.
