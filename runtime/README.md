# Menata Runtime

> **A metadata-driven application runtime for interpreting Business Knowledge.**

Business Knowledge alone does not create applications.

Applications require interpretation.

Pages.

Navigation.

Interaction.

Services.

Integration.

Deployment.

These concerns belong to the runtime.

Menata Runtime exists to interpret Business Knowledge into complete, living applications.

Applications are not generated.

Applications are interpreted.

Applications evolve by changing metadata rather than rewriting source code.

## Why Menata Runtime?

Business Knowledge should not be coupled to application implementation.

The same Business Knowledge should be capable of producing:

- internal business applications,
- portals,
- dashboards,
- APIs,
- AI agents,
- mobile experiences,
- future interaction models.

The challenge is no longer expressing Business Knowledge.

The challenge is interpreting it consistently.

Menata Runtime exists to bridge that gap.

## Architecture

```text
Business Reality
        │
        ▼
Business Knowledge
        │
        ▼
Menata Language
        │
        ▼
Menata Runtime
        │
        ▼
Applications
```

Business Reality is where work happens.

Business Knowledge explains how work is performed.

Menata Language expresses that knowledge.

Menata Runtime interprets the language into running applications.

Applications become living representations of Business Knowledge.

## Vision

We believe every Business Knowledge deserves implementation.

Applications should:

- evolve by changing metadata,
- remain independent from implementation technology,
- be interpreted rather than generated,
- be composable,
- support continuous evolution.

The runtime evolves.

Business Knowledge remains stable.

## Long-term Vision

Menata Runtime aims to become a universal metadata-driven application platform.

A single runtime should be capable of hosting:

- one application,
- hundreds of applications,
- thousands of applications,

from the same runtime environment.

Applications are defined by metadata.

The runtime interprets that metadata into complete application experiences.

As technologies evolve, only the runtime evolves.

Business Knowledge remains unchanged.
